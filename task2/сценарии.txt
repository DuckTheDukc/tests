Тест 1: Проверка создания новой книги и обработки дубликата

Цель тестирования: Убедится, что книги добавляются правильно, и при добавлении дубликата возвращается ошибка.

Предпосылки:
База данных книг пуста

Шаги тестирования:
Добавить новую книгу, не занятую пользователем
Ожидаемый результат: В списке доступных книг появляется новая книга

добавить книгу, с таким же названием
Ожидаемый результат: Дубликат не добавится в список, выведется ошибка "такая книга уже есть"


Тест 2: Проверка выдачи и возврата книги пользователю
Цель тестирования: Убедиться, что статус книги при выдаче и возврате обрабатывается корректно

предпосылки: 
В базе данных есть пользователь "Иван" и книга "Преступление и наказание"

Шаги тестирования:
Выдать книгу ползователю с именем "Иван"
Ожидаемый результат: Статус книги меняется на неактивную, владелец книги меняется на "Иван",
Книга добавляется в список книг пользователя.

сделать возврат книги
Ожидаемый результат: Статус книги меняется на активную, владелец книги меняется на "None",
Книга удаляется из списка книг пользователя.

Тест 3: Проверка переименования книги, которая находится у пользователя

Цель тестирования: Проверить, что при переименовании книги, у пользователя
в списке книг название обновится, а старая запись удалится из словаря.

предпосылки:
В базе данных есть пользователь "Иван", и книга "СтароеНазвание", книга "СтароеНазвание" неактивна и её владелец "Иван",
книга "СтароеНазвание" есть в списке книг "Ивана" 

Шаги тестирования:
Поменять название книги "СтароеНазвание" на "НовоеНазвание"
Ожидаемый результат: Название книги поменяется на новое, старой книги больше нет в базе данных и в списке книг пользователя.


Тест 4: Проверка смены имени пользователя, у которого есть книги
Цель тестирования: Проверить, что владелец у книги обновится на новое имя при смене имени пользователя.

предпосылки:
В базе данных есть пользователь "Иван", и книга "название", книга "название" неактивна и её владелец "Иван",
книга "название" есть в списке книг "Ивана" 

Шаги тестирования:
Поменять имя пользователя "Иван" на "Иван Иванов"
Ожидаемый результат: Имя пользователя поменяется на новое, старого имени больше нет в базе данных и в поле владельца книги,
а книга есть в листе книг пользователя.

Тест 5: Проверка попытки взять уже занятую книгу
Цель тестирования: Проверить, что второй пользователь получит ошибку при попытке взять книгу, которая уже выдана.

Предпосылки: 
В базе данных есть пользователи "Иван" и "Макс", и книга "РедкаяКнига"

Шаги тестирования:
Выдать книгу ползователю с именем "Макс"
Ожидаемый результат: Статус книги меняется на неактивную, владелец книги меняется на "Макс",
Книга добавляется в список книг пользователя.

Выдать книгу ползователю с именем "Иван"
Ожидаемый результат: Статус книги не меняется, владелец не меняется, пользователю выдаст ошибку "Книга уже взята"
